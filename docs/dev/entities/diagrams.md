# Diagrams

Per-project Visio-like diagram editor. Users create typed diagrams (network, org chart, architecture, etc.) on an xyflow canvas, edit them via chat, and drag nodes from a rich library.

## Tables

| Table | Purpose |
|-------|---------|
| `diagrams` | One row per diagram: `project`, `name`, `diagram_type`, `content_json`, `thumbnail`, soft-delete |
| `diagram_node_library` | Seeded global + per-project custom node type definitions |

`content_json` stores xyflow-compatible JSON: `{ nodes: Node[], edges: Edge[] }`.

## Diagram types

`network` · `flowchart` · `orgchart` · `architecture` · `er` · `sequence` · `mindmap` · `class` · `bpmn` · `custom`

## Node library

- **Seeded nodes** (`is_seeded = 1`, `project IS NULL`) — 80+ entries across all 10 types, inserted once at boot by `ensureSeededLibrary`.
- **Custom nodes** (`is_seeded = 0`, `project = X`) — added by users or generated by LLM via the "Add via AI" button.

List query: `WHERE is_seeded = 1 OR project = ?` combines both.

## LLM modes

| Mode | Route | Prompt key | Session |
|------|-------|-----------|---------|
| Generate | `POST /api/diagrams/:id/generate` | `diagram.generate` | hidden |
| Edit | `POST /api/diagrams/:id/edit` | `diagram.edit` | hidden |
| Ask | `POST /api/diagrams/:id/ask` | — (standard agent) | quick-chat |
| Node generate | `POST /api/projects/:p/diagrams/library/generate` | `diagram.node.generate` | hidden |

All SSE modes stream `content` events; the frontend extracts the `\`\`\`json\`\`\`` block and updates the canvas.

## HTTP routes

```
GET    /api/projects/:p/diagrams          list
POST   /api/projects/:p/diagrams          create
GET    /api/diagrams/:id                  get
PATCH  /api/diagrams/:id                  update (name, contentJson, thumbnail, description)
DELETE /api/diagrams/:id                  soft-delete
POST   /api/diagrams/:id/generate         SSE — initial LLM layout
POST   /api/diagrams/:id/edit             SSE — chat-driven edit
POST   /api/diagrams/:id/ask              quick-chat
GET    /api/projects/:p/diagrams/library  list library (query: ?type=)
POST   /api/projects/:p/diagrams/library  create custom node
DELETE /api/diagrams/library/:id          delete custom node (seeded nodes blocked)
POST   /api/projects/:p/diagrams/library/generate  SSE — LLM node generation
```

## Frontend

Entry: `web/src/tabs/DiagramsTab.tsx`

Sub-components under `web/src/tabs/diagrams/`:
- `DiagramCanvas.tsx` — xyflow wrapper with drop handler
- `DiagramNode.tsx` — single custom node type (all shapes via CSS)
- `DiagramNodeLibrary.tsx` — left panel with drag items + "Add via AI"
- `DiagramComposer.tsx` — Edit/Question mode composer
- `DiagramNewDialog.tsx` — create modal with type grid + intent field

`localStorage` keys:
- `bunny.activeDiagram.<project>` — last opened diagram id
- `bunny.diagramLibrary.open.<project>` — library panel open state (future)

## Trash

Registered as `"diagram"` via `registerTrashable`. Restores cleanly via the generic trash admin.
